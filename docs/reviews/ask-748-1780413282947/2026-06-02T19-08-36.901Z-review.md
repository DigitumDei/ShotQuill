## Executive Summary

This branch introduces a comprehensive overhaul of the photo-editing request model: it adds four new fields (userRefinement, subjectDescription, targetPlatform, maskRegion) to PhotoEditRequest, replaces the EditIntent enum with seven new values, adds platform-preset-aware AI prompt enrichment, and wires up full SQL persistence for all new fields. Originally-raised blocking concerns about missing schema migrations and EditIntent backward-compatibility were invalidated by the critiques — the ShotQuill schema and EditIntent enum are both introduced for the first time relative to main (version 1, greenfield), so no upgrade path is required. After debunking those false alarms, no confirmed blocking issues remain; however, the maskRegion feature is entirely dead end-to-end (persisted and validated but never injected into the AI call), the mandatory PhotoEditRequest.prompt field lacks the blank validation that optional sibling fields have, and MaskBounds.Normalized uses strict float equality that real edge-sum values can spuriously violate. The branch is mergeable but ships incomplete infrastructure and several test gaps.
