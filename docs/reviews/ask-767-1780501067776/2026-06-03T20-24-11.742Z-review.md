## Executive Summary

This branch introduces persisted active media selection to PostDraft, allowing users to designate an edited photo as the 'selected' asset after a successful photo edit pipeline run. The central correctness defect — agreed upon by all three reviewers — is that `primaryMediaAsset()` only searches `mediaItems` for `selectedMediaAssetId`, but the pipeline stores edited assets exclusively in `photoEditResults`. The selection is therefore silently dropped for every downstream pipeline call (`VisionDescriptionAnalyzer`, `PostTextGenerationPipeline`), meaning the feature is effectively a no-op at the domain level. A second blocker is a duplicate `val originalAsset` declaration in the same test-function scope, which produces a Kotlin compile error in `PhotoEditExecutionPipelineTest`. The verdict is `revise`: these two issues must be fixed before the branch is mergeable, and the screen-level wiring gap means the user-facing feature is also not yet delivered.
