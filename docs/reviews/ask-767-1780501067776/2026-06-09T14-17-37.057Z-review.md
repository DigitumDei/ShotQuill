## Executive Summary

This branch introduces a photo editing pipeline with AI integration, adding PhotoEditExecutionPipeline, a selected_media_asset_id FK in the database schema, and ViewModel support for selecting between original and edited photos. The newer pipeline paths (generatePostText, editPhotoWithAi) are correctly wired to real providers and are well-tested. The primary consensus finding is that the standalone 'Analyze Vision' UI button remains wired to FakeManualDraftAiProvider in production — the ViewModel constructor never receives a real provider, so cold-start presses of that button persist fabricated vision descriptions. A related correctness bug in the same legacy path allows a cached original-photo vision description to be incorrectly reused for a newly selected edited photo without checking media asset identity. These two issues are user-visible and affect data integrity, warranting revision before merge.
